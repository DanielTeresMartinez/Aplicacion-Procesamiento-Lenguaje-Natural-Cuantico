# Extracting Key Points for Presentations

## Objective
Extract the main concepts and ideas from a specific chapter or section of a LaTeX or Markdown document and summarize them into concise, slide-friendly bullet points. The goal is to provide a clean structure with minimal text, suitable for presentations where the speaker expands on the details and relies on visual diagrams or images.

## Instructions
1. **Analyze the Request**: Identify which chapter, section, or topic the user wants to extract key points from, and the specific slides or layout they are designing (e.g., columns, split screens, comparisons).
2. **Read the Content**: Locate and read the relevant source files (e.g., `.tex` or `.md`) using the file viewing tools.
3. **Extract Core Concepts**:
    - Avoid paragraphs or long blocks of text.
    - Condense sentences into punchy, high-impact bullet points (max 10-12 words per bullet).
    - Maintain academic and technical rigor while ensuring readability from a distance.
4. **Structure the Presentation Ideas**:
    - Group bullet points by slide titles and visual sections (e.g., Left Column, Right Column).
    - Provide suggestions for visual elements (graphs, diagrams, formulas, or images) that can complement the text to make the slide dynamic.
5. **Language Requirement**: The extracted key points must be in **Spanish** (or the language specified by the user), while these skill instructions are maintained in English.
